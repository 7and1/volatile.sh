Our Mission

volatile.sh exists to provide a truly private way to share sensitive information online. We believe privacy is a fundamental right, and the best way to protect user data is to never have access to it in the first place.

Traditional secure messaging services still require you to trust the service provider. volatile.sh eliminates that trust requirement through zero-knowledge architecture‚Äîthe server literally cannot access your data, even if compelled by law or hacked.

Technology Stack

volatile.sh is built on modern, cloud-native infrastructure designed for security, performance, and reliability.

Cloudflare Workers

Edge compute platform with automatic DDoS protection and global distribution.

Durable Objects

Strongly consistent storage with automatic expiration and transactional operations.

AES-256-GCM

Military-grade encryption with authenticated encryption (AEAD) for integrity.

Web Crypto API

Native browser cryptography for secure client-side encryption.

Zero-Knowledge Architecture

The core innovation of volatile.sh is our zero-knowledge design. Here's how it works:

  1. Key Generation: Your browser generates a random AES-256-GCM key using Web Crypto API
  2. Client Encryption: Your secret is encrypted locally before any data leaves your device
  3. Fragment Isolation: The decryption key is embedded in the URL fragment (#key), which browsers never send to servers per RFC 3986
  4. Ephemeral Storage: Only encrypted ciphertext is stored in RAM-only Durable Objects, with automatic expiration
  5. Burn After Reading: Secrets are deleted immediately upon retrieval, leaving no trace

This architecture means we technically cannot decrypt your secrets, even if we wanted to. The encryption key never reaches our servers.

Open Source Philosophy

volatile.sh is 100% open source under the MIT License. This means:

  • You can review every line of code for security vulnerabilities
  • You can run your own instance on your own Cloudflare account
  • You can modify the code to fit your needs
  • You can contribute improvements back to the community

We believe security tools must be transparent. Closed-source "secure" messaging requires blind trust in the provider. Open source allows independent verification of our security claims.

Why Cloudflare Workers?

We chose Cloudflare Workers as our platform for several reasons:

  • Edge Performance: Code runs in data centers worldwide, minimizing latency
  • Automatic Scaling: No server management‚ÄîCloudflare handles everything
  • DDoS Protection: Built-in protection against attacks at the edge
  • Always-on TLS: HTTPS is enforced by default, required for Web Crypto API
  • Free Tier: Generous free tier makes self-hosting accessible to everyone

Project History

volatile.sh was created to address a gap in the market: most "secure" note-sharing services are either closed-source (requiring blind trust) or complex to self-host. We wanted to build something that was:

  • Truly zero-knowledge (server cannot access data)
  • Fully open source (auditable by anyone)
  • Easy to self-host (single command deployment)
  • Free to use (no premium tiers or hidden costs)

The project launched in December 2024 and is actively maintained by the open-source community.

Security & Audits

While we have not yet undergone a professional security audit, we follow security best practices:

  • Minimal dependencies to reduce attack surface
  • Regular dependency vulnerability scanning
  • Static code analysis with ESLint and TypeScript strict mode
  • Comprehensive test coverage
  • Manual code review before each release
